6.1 FA5 Ethiopia 2026 - Genomic Analysis Exercise

7 Exercise: Exploring Trypanosome Genomic Variation

7.1 Objective

Students will perform a full genomic analysis pipeline from raw FASTQ reads to population structure, including mapping, variant calling, phylogenetics, PCA, ADMIXTURE, and IBD network analysis. This exercise emphasizes understanding each step and interpreting results.

7.2 Dataset

  • Subset of Trypanosome FASTQ files: *_R1.fastq.gz and *_R2.fastq.gz
  • Reference genome: L. donovani BPK282.fasta

8 Part 1: Quality Control

8.1 Task 1.1

Run FastQC and MultiQC on raw FASTQ files.

8.1.1 Hint

mkdir -p results/fastqc/
fastqc data/*.fastq.gz -o results/fastqc/
multiqc results/fastqc/ -o results/fastqc/

8.1.2 Questions

  1. Which samples show low-quality scores?
  2. Are there adapters that need trimming?

9 Part 2: Mapping and BAM Processing

9.1 Task 2.1

Align reads to the reference genome using BWA-MEM.

9.1.1 Hint

bwa mem -R "@RG\tID:sample\tSM:sample\tPL:ILLUMINA" -t 4 ref_genome.fasta R1.fastq.gz R2.fastq.gz | samtools sort -o sample.bam

9.2 Task 2.2

Filter BAMs by mapping quality, remove duplicates, and keep only properly paired reads.

9.2.1 Hint

  • Mapping quality: samtools view -q 30
  • Remove duplicates: picard MarkDuplicates REMOVE_DUPLICATES=true
  • Proper pairs: samtools view -f 0x2

9.2.2 Questions

  1. How many reads remain after filtering? Check with samtools flagstat.
  2. Which step caused the largest read loss?

10 Part 3: Variant Calling

10.1 Task 3.1

Use GATK HaplotypeCaller to generate GVCFs for each sample.

10.1.1 Hint

gatk HaplotypeCaller -R ref_genome.fasta -I sample.proper.bam -O sample.g.vcf.gz -ERC GVCF

10.2 Task 3.2

Combine GVCFs and run GenotypeGVCFs to get a joint VCF.

10.3 Task 3.3

  1. Ensure only high-quality SNPs are retained (QUAL > 30).
  2. Confirm variants are biallelic.

10.3.1 Questions

  1. How many SNPs pass your filters?
  2. Why do we keep only biallelic SNPs?

11 Part 4: Phylogenetic Analysis

11.1 Task 4.1

Convert the filtered VCF file into an alignment format suitable for phylogenetic tree construction (FASTA or PHYLIP).

11.1.1 Hint

  • Use a tool that converts VCF to sequence alignment format.
  • Make sure to specify:
    • The input VCF file
    • The output alignment format (FASTA or PHYLIP)
    • The desired output filename

11.2 Task 4.2

Build a phylogenetic tree using a Maximum Likelihood approach. You can start with using the IQ-tree software, as this is the easiest to handle, i.e. not much parameters to define yourself

11.2.1 Hint

Look for options such as: - -m MFP (ModelFinder Plus) - -bb for ultrafast bootstrap - -alrt for SH-aLRT support - -nt AUTO for automatic threading - -s to specify the alignment file

11.2.2 Questions

  1. Which clusters are strongly supported (bootstrap >70)?
  2. Compare the tree topology with PCA results (later).

11.3 Task 4.3

Infer a rapid tree using an approximate method.

11.3.1 Subtasks

  1. Run FastTree on the same alignment.
  2. Compare topology with the IQ-TREE result.
  3. Identify poorly supported branches.

11.3.2 Hint (Think in Terms of Flags)

Consider: - -gtr for nucleotide substitution model - -gamma for rate heterogeneity - -nt for nucleotide data

12 Part 5: PCA Analysis

12.1 Task 5.1

Convert the filtered VCF to PLINK format and perform quality control filtering.

12.1.1 Hint

  • Convert VCF to binary PLINK format.
  • Apply SNP-level filters:
    • Remove SNPs with high missingness.
    • Remove SNPs with low minor allele frequency (MAF).
  • Apply individual-level filtering:
    • Remove individuals with high missingness.

12.2 Task 5.2

Perform LD pruning and compute principal components.

12.2.1 Hint

  • Perform linkage disequilibrium pruning using a sliding window approach.
  • Extract the pruned SNP set.
  • Run PCA and request multiple principal components (e.g., 10 PCs).

12.2.2 Questions

  1. How do samples cluster along PC1 and PC2?
  2. Do clusters correspond to metadata (country, strain)?
  3. Compare your results with the PCA analysis

13 Part 6: ADMIXTURE Analysis

13.1 Task 6.1

Run ADMIXTURE for several values of K and determine the optimal number of ancestral populations.

13.1.1 Hint

  • Use the LD-pruned PLINK dataset as input (.bed file).
  • Enable cross-validation.
  • Run multiple K values (e.g. 2–10).
  • Capture the output in log files.
  • Extract the CV error from each run. (using grep CV)

13.2 Task 6.2

Visualize ancestry proportions in R using stacked barplots.

13.2.1 Questions

  1. Which K has the lowest CV error?
  2. Are there admixed individuals? Which populations contribute to admixture?

14 Part 7: IBD Network Analysis

14.1 Task 7.1

Compute pairwise Identity-By-Descent (IBD) estimates.

14.1.1 Hint

  • Use the pruned PLINK dataset.
  • Compute pairwise relatedness estimates.
  • Identify the column corresponding to PI_HAT.

14.2 Task 7.2

Construct an IBD network in R.

14.2.1 Hint

  • Filter pairs above a meaningful PI_HAT threshold (e.g., >0.125).
  • Build a network graph where:
    • Nodes represent samples
    • Edges represent relatedness
    • Edge weight reflects PI_HAT

14.2.2 Questions

  1. Which samples are most closely related?
  2. Do IBD clusters correspond to PCA or phylogenetic clusters?

15 Tips

  • Check intermediate files with samtools flagstat, bcftools stats, or plink --freq.
  • Use consistent file naming: _rmdup.proper.bam, _filtered.vcf.gz, _QC.pruned.
  • Visualize results at each step: PCA scatterplots, ADMIXTURE barplots, phylogenetic trees.
  • Phylogenetics, PCA, ADMIXTURE, and IBD complement each other — each provides different insight into population structure.
